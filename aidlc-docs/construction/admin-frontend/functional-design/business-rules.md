# Business Rules - Admin Frontend (Unit 3)

## BR-01. 인증 규칙
| ID | 규칙 |
|----|------|
| BR-01-1 | localStorage token 없으면 LoginPage 강제 이동 |
| BR-01-2 | JWT exp 만료 시 자동 logout + LoginPage 이동 |
| BR-01-3 | 인증 필요 페이지 접근 시 token 검증 후 리다이렉트 |

## BR-02. 주문 상태 변경 규칙
| ID | 규칙 |
|----|------|
| BR-02-1 | 상태 전이는 단방향: pending → preparing → completed |
| BR-02-2 | completed 상태에서는 상태 변경 버튼 비활성화 |

## BR-03. 주문/테이블 삭제 규칙
| ID | 규칙 |
|----|------|
| BR-03-1 | 주문 삭제 전 반드시 ConfirmDialog 표시 |
| BR-03-2 | 이용 완료 처리 전 반드시 ConfirmDialog 표시 |

## BR-04. 메뉴 관리 검증
| ID | 규칙 | 에러 메시지 |
|----|------|------------|
| BR-04-1 | 메뉴명 필수 | "메뉴명을 입력해주세요" |
| BR-04-2 | 가격 필수 + 양수 | "올바른 가격을 입력해주세요" |
| BR-04-3 | 카테고리 필수 | "카테고리를 선택해주세요" |

## BR-05. 실시간 업데이트 규칙
| ID | 규칙 |
|----|------|
| BR-05-1 | 신규 주문 수신 시 해당 테이블 카드 3초간 하이라이트 |
| BR-05-2 | SSE 연결 끊김 시 브라우저 자동 재연결 |
| BR-05-3 | 컴포넌트 언마운트 시 SSE 연결 종료 |
