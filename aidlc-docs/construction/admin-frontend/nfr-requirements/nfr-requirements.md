# NFR Requirements - Admin Frontend (Unit 3)

## NFR-01. 성능
| ID | 요구사항 | 목표값 |
|----|---------|--------|
| NFR-01-1 | 대시보드 초기 로드 | 2초 이내 |
| NFR-01-2 | 신규 주문 화면 반영 | 2초 이내 (SSE 이벤트 후) |
| NFR-01-3 | 메뉴 순서 변경 반영 | 즉시 (로컬 상태 선반영) |

## NFR-02. 보안
| ID | 요구사항 |
|----|---------|
| NFR-02-1 | JWT token localStorage 저장, 모든 API 요청에 Bearer 헤더 포함 |
| NFR-02-2 | JWT exp 만료 시 자동 로그아웃 |
| NFR-02-3 | 인증 필요 페이지 접근 시 token 검증 |

## NFR-03. 안정성
| ID | 요구사항 |
|----|---------|
| NFR-03-1 | SSE 연결 끊김 시 브라우저 자동 재연결 |
| NFR-03-2 | API 실패 시 에러 메시지 표시, 데이터 손실 없음 |

## NFR-04. 사용성
| ID | 요구사항 |
|----|---------|
| NFR-04-1 | 신규 주문 시각적 강조 (색상 + 애니메이션) |
| NFR-04-2 | 삭제/완료 처리 전 확인 팝업 |
| NFR-04-3 | 로딩 상태 피드백 |
