# Integration Test Instructions

## 전체 시스템 통합 시나리오

Docker Compose로 전체 서비스 실행 후 아래 시나리오를 순서대로 검증합니다.

---

## 시나리오 1: 초기 설정 → 주문 → 실시간 수신

**사전 준비**:
1. `docker compose up -d` 실행
2. 관리자 계정 생성 (`POST /auth/admin/setup`)
3. 테이블 설정 (관리자 UI → 테이블 관리에서 테이블 비밀번호 설정)

**검증 단계**:

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 관리자 UI 로그인 | DashboardPage 표시 |
| 2 | 고객 UI SetupPage에서 테이블 설정 | MenuPage 이동 |
| 3 | 고객이 메뉴 2개 장바구니 담기 | 장바구니 버튼에 수량/금액 표시 |
| 4 | 고객이 주문 확정 | 주문 번호 표시 → 5초 후 MenuPage |
| 5 | 관리자 대시보드 확인 | 2초 이내 NEW 배지 + 주문 표시 |
| 6 | 관리자가 주문 상태 → 준비중 변경 | 고객 주문내역 화면에서 실시간 반영 |
| 7 | 관리자가 주문 상태 → 완료 변경 | 고객 화면 "완료" 표시 |

---

## 시나리오 2: 테이블 세션 관리

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 고객이 여러 주문 생성 | 대시보드에 누적 표시 |
| 2 | 관리자 → 이용 완료 처리 | 해당 테이블 주문 초기화 |
| 3 | 고객 UI 새로고침 | 주문 내역 비어있음 (새 세션) |
| 4 | 관리자 → 과거 내역 조회 | 이전 세션 주문 목록 표시 |

---

## 시나리오 3: 메뉴 관리 → 고객 반영

| 단계 | 액션 | 기대 결과 |
|------|------|----------|
| 1 | 관리자가 새 메뉴 등록 (이미지 포함) | 메뉴 목록에 표시 |
| 2 | 고객 UI 새로고침 | 새 메뉴 카드 표시, 이미지 로드 |
| 3 | 관리자가 메뉴 순서 드래그 변경 | 고객 UI 새로고침 시 변경된 순서 반영 |
| 4 | 관리자가 메뉴 삭제 | 고객 UI 새로고침 시 메뉴 사라짐 |
