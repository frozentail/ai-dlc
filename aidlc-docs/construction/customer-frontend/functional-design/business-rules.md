# Business Rules - Customer Frontend (Unit 2)

## BR-01. 인증 규칙

| ID | 규칙 | 처리 |
|----|------|------|
| BR-01-1 | localStorage에 자격증명 없으면 SetupPage로 강제 이동 | App.jsx 라우트 가드 |
| BR-01-2 | 자동 로그인 실패 시 SetupPage로 이동 + 에러 표시 | AuthContext |
| BR-01-3 | 인증 필요 페이지 접근 시 token 없으면 SetupPage 리다이렉트 | PrivateRoute |

## BR-02. 초기 설정 검증

| ID | 규칙 | 에러 메시지 |
|----|------|------------|
| BR-02-1 | 매장 식별자 필수 | "매장 식별자를 입력해주세요" |
| BR-02-2 | 테이블 번호 필수 | "테이블 번호를 입력해주세요" |
| BR-02-3 | 비밀번호 필수 | "비밀번호를 입력해주세요" |

## BR-03. 장바구니 규칙

| ID | 규칙 |
|----|------|
| BR-03-1 | 동일 메뉴 추가 시 수량 증가 (중복 항목 없음) |
| BR-03-2 | 수량 1에서 감소 시 항목 삭제 |
| BR-03-3 | 수량은 항상 1 이상 (0 불가) |
| BR-03-4 | 장바구니 변경 시 즉시 localStorage 동기화 |
| BR-03-5 | 앱 초기화 시 localStorage에서 장바구니 복원 |

## BR-04. 주문 규칙

| ID | 규칙 |
|----|------|
| BR-04-1 | 장바구니 비어있으면 OrderConfirmPage 진입 불가 → MenuPage 리다이렉트 |
| BR-04-2 | 주문 성공 후 5초 뒤 자동으로 장바구니 비우고 MenuPage 이동 |
| BR-04-3 | 주문 실패 시 장바구니 유지 (데이터 손실 없음) |
| BR-04-4 | 주문 진행 중 중복 제출 방지 (isLoading 상태로 버튼 비활성화) |

## BR-05. 실시간 업데이트 규칙

| ID | 규칙 |
|----|------|
| BR-05-1 | SSE는 현재 세션의 테이블 ID 기준으로 연결 |
| BR-05-2 | SSE 이벤트 수신 시 해당 주문만 상태 업데이트 (전체 재로드 없음) |
| BR-05-3 | SSE 연결 끊김 시 브라우저 기본 재연결 동작에 위임 |
| BR-05-4 | 컴포넌트 언마운트 시 SSE 연결 종료 |

## BR-06. 네비게이션 규칙

| ID | 규칙 |
|----|------|
| BR-06-1 | 메뉴 화면이 기본(홈) 화면 (`/`) |
| BR-06-2 | 주문 확정 후 5초 타이머 완료 시 자동으로 `/`로 이동 |
| BR-06-3 | 주문 확인 화면에서 뒤로가기 시 장바구니 유지 |
