### Question 1
User Stories 구성 방식을 어떻게 할까요?

A) Feature-Based - 기능(메뉴 조회, 주문 생성, 주문 모니터링 등) 중심으로 구성
B) User Journey-Based - 고객/운영자의 사용 흐름 순서대로 구성
C) Persona-Based - 고객용 스토리 묶음 / 운영자용 스토리 묶음으로 분리
D) 기타 (Other - [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: A

### Question 2
Story 세분화 수준은 어느 정도로 할까요?

A) 큰 단위 (Epic 수준) - 예: "고객은 메뉴를 주문할 수 있다"
B) 중간 단위 (Feature 수준) - 예: "고객은 장바구니에 메뉴를 추가/삭제할 수 있다"
C) 작은 단위 (Task 수준) - 예: "고객은 장바구니에서 메뉴 수량을 증가시킬 수 있다"
D) 기타 (Other - [Answer]: 태그 뒤에 설명해 주세요)

[Answer]: C
