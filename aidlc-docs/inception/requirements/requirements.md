# 테이블 오더 애플리케이션 요구사항 문서

## 인텐트 분석 요약

- **사용자 요청**: 테이블 오더 애플리케이션 신규 개발
- **요청 타입**: New Project (Greenfield)
- **범위 추정**: System-wide (고객 UI, 관리자 UI, Backend API, DB 전체)
- **복잡도 추정**: Complex (다중 사용자 타입, 실시간 통신, 세션 관리 포함)

| 영역 | 기술 |

## 기능 요구사항

### FR-01. 고객용 기능

#### FR-01-1. 테이블 자동 로그인 및 세션 관리
- 관리자가 1회 초기 설정: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력
- 설정 정보 로컬 스토리지에 저장
- 이후 페이지 접근 시 저장된 정보로 자동 로그인
- 16시간 세션 유지

#### FR-01-2. 메뉴 조회 및 탐색
- 메뉴 화면이 기본(홈) 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카드 형태 레이아웃, 터치 친화적 버튼 (최소 44x44px)
- 카테고리 간 빠른 이동

#### FR-01-3. 장바구니 관리
- 메뉴 추가/삭제, 수량 조절 (증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- 로컬 스토리지에 저장 (새로고침 후에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-01-4. 주문 생성
- 주문 내역 최종 확인 화면
- 주문 확정 버튼
- 주문 성공 시: 주문 번호 표시 (5초) → 장바구니 비우기 → 메뉴 화면 자동 리다이렉트
- 주문 실패 시: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 식별자, 테이블 식별자, 메뉴 목록(메뉴명/수량/단가), 총 금액, 세션 ID

#### FR-01-5. 주문 내역 조회
- 현재 테이블 세션의 주문만 표시
- 주문 시간 순 정렬
- 주문별 정보: 주문 번호, 주문 시각, 메뉴 및 수량, 금액, 주문 상태
- 주문 상태 실시간 업데이트 (SSE 또는 폴링)
- 페이지네이션 또는 무한 스크롤

---

### FR-02. 관리자용 기능

#### FR-02-1. 매장 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- JWT 토큰 기반 인증, 16시간 세션 유지
- 브라우저 새로고침 시 세션 유지
- 16시간 후 자동 로그아웃
- bcrypt 비밀번호 해싱
- 로그인 시도 횟수 제한
- 초기 관리자 계정: 별도 초기 설정 API 제공

#### FR-02-2. 실시간 주문 모니터링
- SSE 기반 실시간 주문 수신 (2초 이내 표시)
- SSE 연결 끊김 시 브라우저 기본 자동 재연결
- 그리드/대시보드 레이아웃 (테이블별 카드)
- 각 테이블 카드: 총 주문액, 최신 주문 n개 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- 신규 주문 시각적 강조 (색상 변경, 애니메이션)
- 테이블별 필터링

#### FR-02-3. 테이블 관리
- 테이블 초기 설정: 테이블 번호, 비밀번호 설정, 16시간 세션 생성
- 주문 삭제 (직권 수정): 확인 팝업 → 즉시 삭제 → 총 주문액 재계산
- 테이블 이용 완료 처리: 확인 팝업 → 세션 종료 → 주문 내역 과거 이력 이동 → 현재 주문 목록/총액 리셋
- 과거 주문 내역 조회: 테이블별 과거 주문 목록 (시간 역순), 날짜 필터링

#### FR-02-4. 메뉴 관리
- 메뉴 CRUD (등록/조회/수정/삭제)
- 메뉴 필드: 메뉴명, 가격, 설명, 카테고리, 이미지 (파일 업로드, 리사이징 없음)
- 카테고리별 조회
- 메뉴 노출 순서 조정: 드래그 앤 드롭
- 필수 필드 검증, 가격 범위 검증

## 비기능 요구사항

### NFR-01. 성능
- 실시간 주문 수신: 2초 이내
- 메뉴 조회 응답: 1초 이내

### NFR-02. 보안
- JWT 토큰 기반 인증
- bcrypt 비밀번호 해싱
- 로그인 시도 횟수 제한
- 민감 정보 환경 변수 관리

### NFR-03. 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 카드 형태 메뉴 레이아웃
- 명확한 시각적 계층 구조

### NFR-04. 안정성
- SSE 연결 끊김 시 자동 재연결
- 장바구니 로컬 스토리지 유지 (새로고침 후에도 유지)
