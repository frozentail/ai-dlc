# User Stories - 테이블 오더 애플리케이션

- **구성 방식**: Feature-Based
- **세분화 수준**: Task 수준 (작은 단위)
- **Acceptance Criteria 형식**: Given-When-Then

---

## Feature 1: 테이블 자동 로그인 및 세션 관리

### US-01. 테이블 초기 설정
**As a** 매장 운영자,  
**I want to** 태블릿에 매장 식별자, 테이블 번호, 비밀번호를 1회 입력하고 저장하고 싶다,  
**So that** 이후 고객이 별도 로그인 없이 즉시 주문할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 태블릿에 저장된 로그인 정보가 없고
When 운영자가 매장 식별자, 테이블 번호, 비밀번호를 입력하고 저장하면
Then 입력 정보가 로컬 스토리지에 저장되고
     자동 로그인이 활성화된다
```

AC2:
```
Given 필수 입력값(매장 식별자, 테이블 번호, 비밀번호) 중 하나라도 비어있고
When 저장 버튼을 누르면
Then 에러 메시지가 표시되고 저장되지 않는다
```

---

### US-02. 자동 로그인
**As a** 고객,  
**I want to** 태블릿 화면을 열면 자동으로 로그인되고 싶다,  
**So that** 별도 로그인 절차 없이 바로 메뉴를 볼 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 태블릿에 저장된 로그인 정보가 있고
When 고객이 브라우저를 열거나 페이지를 새로고침하면
Then 저장된 정보로 자동 로그인되고
     메뉴 화면이 표시된다
```

AC2:
```
Given 자동 로그인 시도 중 서버 인증이 실패하면
When 페이지가 로드되면
Then 에러 메시지가 표시되고
     초기 설정 화면으로 이동한다
```

---

## Feature 2: 메뉴 조회 및 탐색

### US-03. 카테고리별 메뉴 목록 조회
**As a** 고객,  
**I want to** 카테고리별로 메뉴 목록을 볼 수 있고 싶다,  
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 고객이 메뉴 화면에 있고
When 화면이 로드되면
Then 카테고리 탭 목록이 표시되고
     첫 번째 카테고리의 메뉴들이 카드 형태로 표시된다
```

AC2:
```
Given 고객이 메뉴 화면에 있고
When 특정 카테고리 탭을 누르면
Then 해당 카테고리의 메뉴들만 표시된다
```

---

### US-04. 메뉴 상세 정보 확인
**As a** 고객,  
**I want to** 메뉴 카드에서 메뉴명, 가격, 설명, 이미지를 확인하고 싶다,  
**So that** 주문 전에 메뉴 정보를 충분히 파악할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 고객이 메뉴 목록을 보고 있고
When 메뉴 카드가 표시되면
Then 메뉴명, 가격, 설명, 이미지가 카드에 표시된다
```

AC2:
```
Given 메뉴에 이미지가 없고
When 메뉴 카드가 표시되면
Then 기본 placeholder 이미지가 표시된다
```

---

## Feature 3: 장바구니 관리

### US-05. 장바구니에 메뉴 추가
**As a** 고객,  
**I want to** 메뉴 카드에서 "추가" 버튼을 눌러 장바구니에 담고 싶다,  
**So that** 주문할 메뉴를 선택해 놓을 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 고객이 메뉴 카드를 보고 있고
When "추가" 버튼을 누르면
Then 해당 메뉴가 장바구니에 1개 추가되고
     장바구니 총 금액이 실시간으로 업데이트된다
```

AC2:
```
Given 장바구니에 이미 동일 메뉴가 있고
When "추가" 버튼을 다시 누르면
Then 해당 메뉴의 수량이 1 증가한다
```

---

### US-06. 장바구니 메뉴 수량 조절
**As a** 고객,  
**I want to** 장바구니에서 메뉴 수량을 증가/감소시키고 싶다,  
**So that** 주문 수량을 원하는 대로 조정할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 장바구니에 메뉴가 1개 이상 있고
When "+" 버튼을 누르면
Then 해당 메뉴 수량이 1 증가하고 총 금액이 업데이트된다
```

AC2:
```
Given 장바구니에 메뉴가 2개 이상 있고
When "-" 버튼을 누르면
Then 해당 메뉴 수량이 1 감소하고 총 금액이 업데이트된다
```

AC3:
```
Given 장바구니에 메뉴가 1개 있고
When "-" 버튼을 누르면
Then 해당 메뉴가 장바구니에서 삭제된다
```

---

### US-07. 장바구니 메뉴 삭제
**As a** 고객,  
**I want to** 장바구니에서 특정 메뉴를 삭제하고 싶다,  
**So that** 원하지 않는 메뉴를 제거할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 장바구니에 메뉴가 있고
When 삭제 버튼을 누르면
Then 해당 메뉴가 장바구니에서 제거되고
     총 금액이 재계산된다
```

---

### US-08. 장바구니 전체 비우기
**As a** 고객,  
**I want to** 장바구니를 한 번에 비우고 싶다,  
**So that** 처음부터 다시 메뉴를 선택할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 장바구니에 메뉴가 있고
When "장바구니 비우기" 버튼을 누르면
Then 장바구니의 모든 메뉴가 삭제되고
     총 금액이 0원으로 표시된다
```

---

### US-09. 장바구니 로컬 저장
**As a** 고객,  
**I want to** 페이지를 새로고침해도 장바구니가 유지되길 원한다,  
**So that** 실수로 새로고침해도 선택한 메뉴를 잃지 않는다.

**Acceptance Criteria**:

AC1:
```
Given 장바구니에 메뉴가 담겨 있고
When 페이지를 새로고침하면
Then 장바구니의 메뉴 목록과 수량이 그대로 유지된다
```

---

## Feature 4: 주문 생성

### US-10. 주문 내역 최종 확인
**As a** 고객,  
**I want to** 주문 확정 전에 주문 내역을 최종 확인하고 싶다,  
**So that** 잘못된 주문을 방지할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 장바구니에 메뉴가 있고
When 주문하기 버튼을 누르면
Then 주문 확인 화면에 메뉴 목록, 수량, 단가, 총 금액이 표시된다
```

---

### US-11. 주문 확정
**As a** 고객,  
**I want to** 주문 확인 화면에서 주문을 확정하고 싶다,  
**So that** 실제 주문이 매장에 전달된다.

**Acceptance Criteria**:

AC1:
```
Given 고객이 주문 확인 화면에 있고
When "주문 확정" 버튼을 누르면
Then 주문이 서버에 전송되고
     주문 번호가 화면에 표시되며
     5초 후 자동으로 메뉴 화면으로 이동하고
     장바구니가 비워진다
```

AC2:
```
Given 고객이 주문 확정 버튼을 눌렀고
When 서버 전송이 실패하면
Then 에러 메시지가 표시되고
     장바구니는 유지된다
```

---

## Feature 5: 주문 내역 조회 (고객)

### US-12. 현재 세션 주문 내역 조회
**As a** 고객,  
**I want to** 현재 테이블 세션의 주문 내역을 확인하고 싶다,  
**So that** 내가 주문한 내용과 상태를 파악할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 고객이 주문 내역 화면에 있고
When 화면이 로드되면
Then 현재 세션의 주문 목록이 시간 순으로 표시되고
     각 주문의 번호, 시각, 메뉴 목록, 금액, 상태가 표시된다
```

AC2:
```
Given 고객이 주문 내역 화면에 있고
When 관리자가 주문 상태를 변경하면
Then 고객 화면의 주문 상태가 실시간으로 업데이트된다
```

AC3:
```
Given 이전 세션(이용 완료 처리된)의 주문이 있고
When 고객이 주문 내역 화면을 보면
Then 이전 세션의 주문은 표시되지 않는다
```

---

## Feature 6: 매장 운영자 인증

### US-13. 관리자 로그인
**As a** 매장 운영자,  
**I want to** 매장 식별자, 사용자명, 비밀번호로 로그인하고 싶다,  
**So that** 내 매장의 관리 화면에 접근할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 로그인 화면에 있고
When 올바른 매장 식별자, 사용자명, 비밀번호를 입력하고 로그인하면
Then JWT 토큰이 발급되고
     관리자 대시보드로 이동한다
```

AC2:
```
Given 운영자가 로그인 화면에 있고
When 잘못된 자격증명을 입력하면
Then 에러 메시지가 표시되고 로그인되지 않는다
```

AC3:
```
Given 운영자가 로그인에 연속으로 5회 실패하면
When 다시 로그인을 시도하면
Then 일정 시간 동안 로그인이 차단된다
```

---

### US-14. 세션 유지 및 자동 로그아웃
**As a** 매장 운영자,  
**I want to** 브라우저를 새로고침해도 로그인 상태가 유지되길 원한다,  
**So that** 매번 다시 로그인하지 않아도 된다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 로그인한 상태에서
When 브라우저를 새로고침하면
Then 로그인 상태가 유지되고 대시보드가 표시된다
```

AC2:
```
Given 운영자가 로그인한 지 16시간이 지났고
When 관리자 화면에 접근하면
Then 자동으로 로그아웃되고 로그인 화면으로 이동한다
```

---

## Feature 7: 실시간 주문 모니터링

### US-15. 테이블별 주문 대시보드 조회
**As a** 매장 운영자,  
**I want to** 테이블별 주문 현황을 그리드 형태로 한눈에 보고 싶다,  
**So that** 어느 테이블에서 주문이 들어왔는지 빠르게 파악할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 대시보드에 있고
When 화면이 로드되면
Then 테이블별 카드가 그리드 형태로 표시되고
     각 카드에 테이블 번호, 총 주문액, 최신 주문 미리보기가 표시된다
```

---

### US-16. 신규 주문 실시간 수신
**As a** 매장 운영자,  
**I want to** 새 주문이 들어오면 2초 이내에 화면에 표시되길 원한다,  
**So that** 주문을 즉시 확인하고 처리할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 대시보드를 보고 있고
When 고객이 주문을 확정하면
Then 2초 이내에 해당 테이블 카드에 신규 주문이 표시되고
     시각적 강조(색상 변경 또는 애니메이션)가 적용된다
```

AC2:
```
Given SSE 연결이 끊겼고
When 브라우저가 자동 재연결을 시도하면
Then 재연결 후 최신 주문 상태가 반영된다
```

---

### US-17. 주문 상세 내역 확인
**As a** 매장 운영자,  
**I want to** 테이블 카드를 클릭해서 전체 주문 메뉴 목록을 보고 싶다,  
**So that** 어떤 메뉴가 주문됐는지 상세하게 확인할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 대시보드에서 테이블 카드를 보고 있고
When 주문 카드를 클릭하면
Then 해당 주문의 전체 메뉴 목록, 수량, 금액, 주문 시각이 표시된다
```

---

### US-18. 주문 상태 변경
**As a** 매장 운영자,  
**I want to** 주문 상태를 대기중 → 준비중 → 완료로 변경하고 싶다,  
**So that** 주문 처리 진행 상황을 관리할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 주문 상세 화면에 있고
When 주문 상태 변경 버튼을 누르면
Then 주문 상태가 다음 단계로 변경되고
     고객 화면에도 실시간으로 반영된다
```

---

## Feature 8: 테이블 관리

### US-19. 주문 삭제 (직권 수정)
**As a** 매장 운영자,  
**I want to** 특정 주문을 삭제하고 싶다,  
**So that** 잘못된 주문을 정정할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 주문 상세 화면에 있고
When 주문 삭제 버튼을 누르면
Then 확인 팝업이 표시된다
```

AC2:
```
Given 확인 팝업이 표시된 상태에서
When 운영자가 삭제를 확인하면
Then 해당 주문이 즉시 삭제되고
     테이블 총 주문액이 재계산된다
```

---

### US-20. 테이블 이용 완료 처리
**As a** 매장 운영자,  
**I want to** 고객이 떠난 테이블을 이용 완료 처리하고 싶다,  
**So that** 새 고객이 이전 주문 내역 없이 새 세션을 시작할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 테이블 관리 화면에 있고
When 이용 완료 버튼을 누르면
Then 확인 팝업이 표시된다
```

AC2:
```
Given 확인 팝업에서 이용 완료를 확인하면
When 처리가 완료되면
Then 현재 세션의 주문 내역이 과거 이력으로 이동되고
     테이블 현재 주문 목록과 총 주문액이 0으로 리셋되며
     새 세션이 시작된다
```

---

### US-21. 과거 주문 내역 조회
**As a** 매장 운영자,  
**I want to** 테이블의 과거 주문 이력을 조회하고 싶다,  
**So that** 이전 고객의 주문 내역을 확인할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 테이블 관리 화면에 있고
When "과거 내역" 버튼을 누르면
Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시되고
     각 주문의 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다
```

AC2:
```
Given 과거 내역 화면에서
When 날짜 필터를 적용하면
Then 해당 날짜 범위의 주문만 표시된다
```

## Feature 9: 메뉴 관리

### US-22. 메뉴 등록
**As a** 매장 운영자,  
**I want to** 새 메뉴를 등록하고 싶다,  
**So that** 고객이 새 메뉴를 주문할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 메뉴 관리 화면에 있고
When 메뉴명, 가격, 설명, 카테고리, 이미지를 입력하고 저장하면
Then 새 메뉴가 등록되고 메뉴 목록에 표시된다
```

AC2:
```
Given 필수 필드(메뉴명, 가격, 카테고리)가 비어있고
When 저장 버튼을 누르면
Then 에러 메시지가 표시되고 저장되지 않는다
```

AC3:
```
Given 가격에 음수 또는 비정상적인 값이 입력되고
When 저장 버튼을 누르면
Then 가격 범위 에러 메시지가 표시된다
```

---

### US-23. 메뉴 수정
**As a** 매장 운영자,  
**I want to** 기존 메뉴 정보를 수정하고 싶다,  
**So that** 메뉴 정보를 최신 상태로 유지할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 메뉴 목록에서 특정 메뉴를 선택하고
When 정보를 수정하고 저장하면
Then 변경된 정보가 즉시 반영되고
     고객 메뉴 화면에도 업데이트된다
```

---

### US-24. 메뉴 삭제
**As a** 매장 운영자,  
**I want to** 메뉴를 삭제하고 싶다,  
**So that** 더 이상 판매하지 않는 메뉴를 제거할 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 메뉴 목록에서 삭제 버튼을 누르면
When 확인 팝업에서 삭제를 확인하면
Then 해당 메뉴가 삭제되고 고객 메뉴 화면에서도 사라진다
```

---

### US-25. 메뉴 노출 순서 조정
**As a** 매장 운영자,  
**I want to** 드래그 앤 드롭으로 메뉴 순서를 변경하고 싶다,  
**So that** 원하는 순서로 메뉴를 고객에게 보여줄 수 있다.

**Acceptance Criteria**:

AC1:
```
Given 운영자가 메뉴 관리 화면에 있고
When 메뉴 카드를 드래그해서 다른 위치에 드롭하면
Then 메뉴 순서가 변경되고
     고객 메뉴 화면에도 변경된 순서가 반영된다
```
